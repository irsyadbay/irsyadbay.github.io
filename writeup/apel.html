SQL Injection pada Website Program Studi TI UKRIM

Irsyad Bayu Kurniawan — Web Security Researcher

Target

https://www.ti.ukrimuniversity.ac.id/berita.php?id_konten=214

Awal Cerita

Semua berawal dari pengamatan sederhana terhadap struktur URL pada halaman berita. Parameter id_konten terlihat seperti identifier biasa, namun menjadi titik awal eksplorasi.

Pengujian dilakukan secara manual tanpa tools otomatis, dengan fokus memahami bagaimana sistem memproses input dari pengguna.

Titik Kecurigaan

Saat nilai parameter dimodifikasi, respons halaman menunjukkan perilaku yang tidak konsisten. Perubahan ini mengindikasikan bahwa input kemungkinan tidak divalidasi dengan baik oleh sistem.

/berita.php?id_konten=214

Analisis Teknis

Pengujian lebih lanjut menunjukkan bahwa input pengguna diproses langsung oleh query backend. Tidak ditemukan mekanisme validasi atau sanitasi yang memadai.

Dalam beberapa kondisi, sistem menampilkan error database yang mengarah pada penggunaan MariaDB serta memperlihatkan sebagian struktur internal (disamarkan):

MariaDB 10.x.x
inf*******_schema
ukr******_coba
ukr******_it
ukr******@localhost

Dampak

Kerentanan ini berpotensi memberikan dampak signifikan terhadap sistem:

  • Akses tidak sah ke data sensitif
  • Manipulasi atau penghapusan data
  • Kemungkinan eskalasi akses

Rekomendasi

  • Menggunakan prepared statement / parameterized query
  • Validasi input bahwa id_konten hanya berupa angka
  • Menonaktifkan error database di environment production

Penutup

Temuan ini menunjukkan bahwa parameter sederhana dapat menjadi titik masuk serangan serius jika tidak divalidasi dengan baik. Pendekatan manual dan analisis mendalam membantu mengidentifikasi celah yang sering terlewat.

Kerentanan telah dilaporkan dan menunggu tindak lanjut dari pihak terkait.
