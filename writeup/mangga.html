SQL Injection pada Situs BKD Maluku Provinsi

Irsyad Bayu Kurniawan — Web Security Researcher

Target

https://bkd.malukuprov.go.id/detail.aspx?id=85

Awal Penemuan

Pengujian dimulai ketika saya melakukan eksplorasi pada halaman publik dari situs BKD Maluku Provinsi. Salah satu parameter yang menarik perhatian adalah id pada endpoint detail konten.

Secara struktur, parameter tersebut terlihat sederhana dan umum digunakan sebagai identifier. Namun, dari sudut pandang keamanan, parameter seperti ini sering kali menjadi titik awal untuk pengujian lebih lanjut.

Proses Pengujian

Pengujian dilakukan secara manual dengan pendekatan observasi terhadap respons sistem. Saya mencoba memahami bagaimana aplikasi memproses input yang diberikan melalui URL.

Saat parameter dimodifikasi, ditemukan adanya perubahan perilaku pada respons server. Hal ini menjadi indikasi awal bahwa input kemungkinan langsung diproses oleh query backend tanpa validasi yang kuat.

Analisis Teknis

Setelah dilakukan beberapa percobaan lanjutan, ditemukan indikasi bahwa sistem menggunakan database berbasis SQL Server. Dalam kondisi tertentu, sistem juga memberikan error yang mengarah pada struktur internal database.

Meskipun informasi yang muncul sangat terbatas dan tidak dieksploitasi lebih jauh, hal tersebut cukup untuk mengonfirmasi adanya potensi SQL Injection.

Payload Pengujian

id=85' AND 4827=(SELECT COUNT(*) FROM sysusers AS sys1,sysusers AS sys2,sysusers AS sys3,sysusers AS sys4,sysusers AS sys5,sysusers AS sys6,sysusers AS sys7) AND 'jenx'='jenx

Dampak Potensial

Jika kerentanan ini dimanfaatkan oleh pihak yang tidak bertanggung jawab, maka dampaknya bisa cukup serius.

  • Pengungkapan data dari database
  • Enumerasi struktur database internal
  • Manipulasi atau modifikasi data
  • Akses tidak sah ke sistem backend
  • Potensi eskalasi ke level administratif

Rekomendasi

Untuk mencegah kerentanan serupa di masa mendatang, beberapa langkah berikut disarankan:

  • Menggunakan prepared statement atau parameterized query
  • Melakukan validasi input secara ketat pada parameter id
  • Menonaktifkan pesan error detail di lingkungan production
  • Melakukan audit keamanan secara berkala
  • Mengimplementasikan Web Application Firewall (WAF)

Penutup

Temuan ini menunjukkan bahwa bahkan parameter sederhana dapat menjadi titik masuk yang berbahaya jika tidak diamankan dengan benar. Pendekatan eksplorasi manual sangat membantu dalam mengidentifikasi celah yang sering terlewat oleh pengembang.

Kerentanan ini telah ditemukan melalui pengujian terbatas dan disusun sebagai bagian dari praktik responsible disclosure.
