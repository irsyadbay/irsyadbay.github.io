SQL Injection pada Website UKRIM

Irsyad Bayu Kurniawan — Bug Hunter

Target

https://www.ti.ukrimuniversity.ac.id/berita.php?id_konten=214

Latar Belakang

Pengujian dimulai dari observasi sederhana terhadap parameter URL pada halaman berita. Parameter id_konten terlihat seperti identifier biasa, namun berpotensi menjadi entry point jika tidak divalidasi dengan benar.

Pendekatan yang digunakan adalah manual testing, dengan fokus memahami bagaimana sistem memproses input dari pengguna.

Identifikasi Awal

Saat parameter dimodifikasi, ditemukan perubahan perilaku pada respon aplikasi. Hal ini menjadi indikasi bahwa input tidak diproses secara aman.

/berita.php?id_konten=214

Analisis Teknis

Pengujian lebih lanjut menunjukkan bahwa input dari pengguna langsung dimasukkan ke dalam query backend tanpa sanitasi yang memadai.

Dalam kondisi tertentu, sistem menampilkan error database yang memberikan petunjuk terkait teknologi yang digunakan.

MariaDB 10.x.x
inf*******_schema
ukr******_coba
ukr******_it
ukr******@localhost

Dampak

  • Akses tidak sah ke data
  • Manipulasi atau penghapusan informasi
  • Enumerasi database
  • Potensi eskalasi akses

Analisis Risiko

SQL Injection merupakan salah satu kerentanan kritikal karena memungkinkan attacker berinteraksi langsung dengan database.

Jika dieksploitasi lebih lanjut, attacker dapat melakukan dump database, bypass authentication, atau mengubah data penting dalam sistem.

Rekomendasi

  • Gunakan prepared statement atau parameterized query
  • Validasi bahwa parameter id_konten hanya berupa integer
  • Nonaktifkan error database di production
  • Lakukan audit keamanan berkala

Kesimpulan

Kerentanan ini menunjukkan bahwa validasi input merupakan aspek penting dalam pengembangan aplikasi web. Parameter sederhana dapat menjadi titik masuk serangan serius jika tidak ditangani dengan benar.

Temuan ini disusun sebagai bagian dari responsible disclosure dan tidak disalahgunakan.
